# Portfolio Website Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and deploy a single-page React + Vite + TypeScript portfolio site for Deepkumar Maheswari to `https://deepmaheswari.github.io`, matching the approved design spec.

**Architecture:** A Vite-scaffolded React/TS SPA. Content (experience, skills, projects, certifications) lives in typed data files under `src/data/`; presentational section components under `src/components/` read that data and render it. A React Three Fiber canvas renders an abstract animated 3D shape in the hero. Static assets (resume PDF, photo) live in `public/`. A GitHub Actions workflow builds and deploys `dist/` to GitHub Pages on every push to `main`.

**Tech Stack:** React 18, Vite, TypeScript, @react-three/fiber, @react-three/drei, three, framer-motion. No test framework — this is a static content site verified via `npm run build` (type-check) and manual browser check (per spec's Testing section), not unit tests.

## Global Constraints

- Repo/site must deploy to `https://deepmaheswari.github.io` — repo name must be exactly `deepmaheswari.github.io` and Vite `base` must be `/`.
- Dark theme: near-black background (`#0a0e14` family), off-white text, cyan/teal accent (`#22d3ee` family).
- Fonts: Inter (body), JetBrains Mono (accent/labels).
- All resume-derived text lives in `src/data/*.ts`, never hardcoded inside component JSX.
- Projects are a single array in `src/data/projects.ts` — adding a project must never require touching component code.
- Static assets referenced by root-relative paths (`/resume.pdf`, `/profile-photo.jpg`) so they resolve correctly under root-site GitHub Pages hosting.
- No backend, no contact form submission — contact via `mailto:` and external profile links only.
- `npm run build` (TypeScript compile + Vite build) must succeed with zero errors before any commit that touches app code.

- [ ] **Step 2: Write src/components/HeroScene.tsx**

```tsx
import { Suspense, useRef, useState, useEffect } from 'react'
import { Canvas, useFrame } from '@react-three/fiber'
import type { Mesh } from 'three'

function SpinningShape() {
  const meshRef = useRef<Mesh>(null)
  const pointer = useRef({ x: 0, y: 0 })

  useEffect(() => {
    const handlePointerMove = (event: PointerEvent) => {
      pointer.current.x = (event.clientX / window.innerWidth) * 2 - 1
      pointer.current.y = (event.clientY / window.innerHeight) * 2 - 1
    }
    window.addEventListener('pointermove', handlePointerMove)
    return () => window.removeEventListener('pointermove', handlePointerMove)
  }, [])

  useFrame((_, delta) => {
    if (!meshRef.current) return
    meshRef.current.rotation.x += delta * 0.15
    meshRef.current.rotation.y += delta * 0.2
    meshRef.current.rotation.x += pointer.current.y * delta * 0.05
    meshRef.current.rotation.y += pointer.current.x * delta * 0.05
  })

  return (
    <mesh ref={meshRef}>
      <icosahedronGeometry args={[1.6, 1]} />
      <meshStandardMaterial color="#22d3ee" wireframe emissive="#0e7490" emissiveIntensity={0.4} />
    </mesh>
  )
}

function hasWebGL(): boolean {
  try {
    const canvas = document.createElement('canvas')
    return !!(window.WebGLRenderingContext && (canvas.getContext('webgl') || canvas.getContext('experimental-webgl')))
  } catch {
    return false
  }
}

export default function HeroScene() {
  const [webglSupported, setWebglSupported] = useState(true)

  useEffect(() => {
    setWebglSupported(hasWebGL())
  }, [])

  if (!webglSupported) {
    return <div className="hero-scene-fallback" />
  }

  return (
    <Canvas camera={{ position: [0, 0, 5], fov: 50 }}>
      <ambientLight intensity={0.4} />
      <pointLight position={[5, 5, 5]} intensity={1.2} color="#22d3ee" />
      <Suspense fallback={null}>
        <SpinningShape />
      </Suspense>
    </Canvas>
  )
}
```

    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}

      - name: Deploy to GitHub Pages
        id: deployment
